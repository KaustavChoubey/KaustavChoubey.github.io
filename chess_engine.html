THE PROGRAM DESCRIPTION:

The program takes the positions of the 2 Rooks and 1 King from the user. The user gives the column number i.e. horizontal box index and the row number i.e. vertical box index as the inputs for each of the pieces. If the King is placed not on the edge of the board i.e. on Row 1 or 8 or on Column 1 or 8 then it is impossible to give the King a checkmate using the 2 Rooks. If the King is placed on Column 1 or 8 then checkmate can be given in the following ways:

  • If the first rook is just below or above the King in the King's column and another rook is in the row of the first rook just next to it, then it is a checkmate. If the other Rook is not next to the first one but in the same row then it can be brought just next to it in 1 step thus checkmating the King. If the other Rook is in the column next to the first one but not on the same row then the King can be given a checkmate in 1 step by bringing the other rook in the row of the first rook.

  • If the first rook is in the column of the King such that King cannot attack that Rook in 1 step(non-target zone) and another rook is in the column next to the King's column such that it is also not in the target zone of the King then it is a checkmate. If the first rook is in its correct place and the other one isn't in the column next to the King's column and is in a different column such that the if that rook is brought straight to the column next to the King's column it won't fall in the target zone of the King then checkmate can be obtained in just 1 step.

  • If the first rook is in the column just next to the King's column in King's target zone and another rook is in the King's column such that it is in the row of the first one then it is a checkmate. If the second Rook is not in the row of the first rook but in the column of the King then the King can be given a checkmate by bringing the first rook to the second rook's row in one step.

    • Similar conditions hold when the King is either in row 1 or in row 8.

The above program checks if these conditions are met and if it is met anywhere the value of the count variable is upgraded from '0' to '1'. There are 2 uses of the count variable. Firstly, it saves the time. If count>0 then the the remaining 'for' loops will remain unaccessed as it unessential after meeting one condition for checkmate and the program will display the output. Secondly, the count variable helps to identify if giving a checkmate in one step is possible or not. If till the end of the program the value of count is 0 then that means it isn't possible to give the King a checkmate in a single step.

Deep Blue - Introduction

Deep Blue is a chess-playing computer under some predefined time controls which was created by the International Business Machines Corporation - American corporation. It became the first system to beat the chess world championship, Garri Kasparow in 1996. Even though the computer did not win other games, it still became a very famous game.

Deep Blue has been constantly improved and at some point it got its name changed to Deeper Blue due to many ameliorations. Another chess game had a place in 1997 and the computer once again won.

Deep Blue - History

The first work on the system has started in 1985 with the ChipTest project at Carnegie Mellon University.Its initial name was Deep Thought but was renamed to Deep Blue in 1989. The American Chess Grandmaster Joel Benjamin was a part of the team hired by IBM that was working on the project.

Al Technique used by chess-playing system

Tree Search is a graphical representation of all the possible moves from the initial stages of the game. And Deep Blue is a system to carry out that method when playing chess. [1]

Deep Blue Algorithm

One of the functions included in the system is the move generation where the computer checks evasion moves, develops certain kinds of attacking moves, supports several search extensions and also calculates all the possible moves and selects one. [2]
